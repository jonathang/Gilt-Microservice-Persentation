Some of the technology pain points
(As of 2009)

  • The amount of traffic at the spike required us to launch 1,000s of ruby processes
  • Postgres was overloaded
  • Routing traffic between ruby processes is hard and was not efficiently solved then
  • Ruby 1.8.* - doesn't really support multi-threading, and the GC was not optimized

Some of the software
development pain points

  • The App code base was huge: 1000 Models and Controllers, 200K lines of code, 100s of jobs
  • Dozens of engineers committing to the same code base without clear ownership
  • Difficult deployments, due to long test/integration cycles

We needed to solve the problem fast

- Invested money - Money we lost, shitty customer experience

Scala Break

Why to switch to Scala from Java

  • Object-Functional Programming
  • Immutability that leads to easier concurrency
  • Great libraries: like Salat, Scalaz
  • Less boilerplate code - e.g Case classes, App
  • Scala's Collections
  • Traits
  • Cake Pattern
  • Console
  • SBT (in scala, release process)
  • Akka
  • No Nulls by default

Why Not to switch to Scala from Java

  • Learning Curve
  • SBT is not that simple
  • Harder debugging in some cases
  • Still early - version upgrades
  • It allows many styles of coding - We are still in the progress of
  • A lot of hard to read methods like: ":+", "++"
  • Can be tricky to integrate with Java code
