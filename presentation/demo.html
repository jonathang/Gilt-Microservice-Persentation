Some of the pain points

  • The amount of traffic at the spike required us to launch 10,000s of ruby processes
  • Routing traffic between ruby processes is hard and was not efficantly solved then
  • Ruby 1.8.* - doesn't really support multi-threading
  • Postgres was overloaded
  • The App code base was huge: 1000 Models and Controllers, 200K lines of code
  • Dozens of engineers commiting to the same code base without clear ownership
  • Difficult deployments, due to long test cycles and difficult merges

We needed to solve the problem fast

Both customer facing and the internal site went down at noon

- Invested money - Money we lost, shitty customer experience

why MSA is good for us?

Schema Evolution Manager - Manage Micro-DBs

https://github.com/gilt/schema-evolution-manager

Principles

  • We manage the schema evolutions as proper source code - Git repo
  • Schema changes are deployed as tar files
  • No rollbacks
  • Schema changes are required to be incremental.

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the background.

Custom Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


                        Reveal.addEventListener( 'customevent', function() {
                        console.log( '"customevent" has fired' );
                        } );

Slide Backgrounds

Set data-background="#007777" on a slide to change the full page background to the given color. All CSS color formats are supported.

Background Transitions
